# Skill Archetype Router

Use this to determine which design guide applies.

## Step 1: Identify Convergence

What does the skill ultimately produce?

- A list of problems → Diagnostic
- A usable artifact → Delivery (provisional — check Step 3)
- A mental state → Stance
- A decision/judgment → Director
- An executed process → Workflow
- A tool output → Pipeline
- Retrieved knowledge → Knowledge
- An understanding/synthesis → Research

## Step 2: Identify Secondary Archetypes

Most skills have more than one. Note them.

## Step 3: Check Capability Center of Gravity

This step prevents artifact-producing skills from being misclassified as Delivery when their true value is elsewhere.

Read the skill as a whole. Does it spend more text on:

- Output format, templates, and artifact structure → Delivery weight
- Judgment, taste, tradeoffs, what-to-keep/what-to-cut → Director weight
- Stance, attitude, suspicion, creative posture → Stance weight
- Domain knowledge, references, retrieval → Knowledge weight

The center of gravity rule:
If a skill produces an artifact BUT spends more than half its substance on judgment/stance/tradeoffs, reclassify primary to Director or Stance, and put Delivery as secondary.

Examples:
- frontend-design produces UI code but its substance is design philosophy, taste, and judgment calls. → Director primary, Delivery secondary.
- high-end-visual-design produces UI code but its substance is banned-pattern rules, persona override, and composition decisions. → Director primary, Stance secondary, Delivery tertiary.
- baoyu-infographic produces an image but its substance is layout/style combinatorics, file management, and backend resolution. → Delivery primary, Workflow secondary.

## Step 4: Check Thickness

Based on base model prior and archetype:

- Diagnostic: strong prior → thin; weak prior → medium
- Delivery: always medium-thick (needs format, iteration)
- Stance: always thin or near-zero (pure posture)
- Director: medium (needs thesis + tradeoffs)
- Workflow: medium-thick (needs stages + recovery)
- Pipeline: thick (needs explicit tool specs)
- Knowledge: medium-thick (needs structure + boundaries)
- Research: thin-medium (needs method, not answers)

## Step 5: Output Classification

Always output:
- Primary Archetype
- Secondary Archetypes (ordered by weight)
- Convergence Target
- Success Metric
- Main Failure Mode
- Misclassification Risk
